In Intro to HTML, I Learned...

How and Why to Validate Code

I learned in the begining of the course that it is important to check your markup through some third party websites. For example, the book had us use the W3C HTML and CSS code validators.

Similarly, it is important check your website in a browser often and make sure that it has the appearance that you want. And, perhaps more importantly, to regularly check your site, all of its pages, and all of its features in several of the most popular web browsers as different browsers interpret instructions differently.

In desktops, the most used browsers according to Wikipedia.com:

  • Google Chrome
  • Internet Explorer
  • Mozilla Firefox
  • Microsoft Edge
  • Safari

How and Why to Use CSS

In this course I also learned how to style different elements in a website. Aside from the built in HTML elements such as emphasis and strong, I've learned how to use Cascading Style Sheets to make my webpages look better than they would without.

On top of styling, I've learned of the three different ways to utilize CSS. For instance, for a limitted amount of CSS on a single webpage, and embedded stylesheet would be apt to handle the amount of styling required. For someone just learning, inline styling my be best as that would allow for better association amongst the elements. And for a large multipage website, a combination of embedded and external stylesheets would be optimal.

About Liquid and Fixed-Width Layouts

On a more advanced note, I learned about making different types of layouts. A Fixed-Width Layout is on which stays the same width. Often times, this means that the widths, paddings, and margins all have specific values represented in pixels.

Also, one could use a Liquid Layout (this website). In this type of layout, widths, paddings, and margins are typically represented with percents. Using a percent to describe the desired width of something in CSS means to take that percentage of the element in which the one being styled is nested.
